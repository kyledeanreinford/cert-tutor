# cert-tutor — Spec

A small, **fully offline** CLI study tool for the **Claude Certified Architect — Foundations (CCA-F)** exam. No API keys, no network calls, no LLM at runtime. Just a curated question bank, a flashcard-style study loop, and a timed mock exam.

## Why it exists

Anthropic Partner Network engineers preparing for the CCA-F want spaced practice against a real-feeling question bank, scored the way the actual exam scores them, with quick links back to the source material when a question stings. Existing flashcard apps don't know about the CCA-F domain weights, blueprint, or the Partner Network learning path.

## The exam being targeted

| Field | Value |
|---|---|
| Name | Claude Certified Architect – Foundations (CCA-F) |
| Provider | Anthropic |
| Questions | 60 |
| Time | 120 minutes |
| Passing | 720 / 1000 (~72%) |
| Format | Multiple choice, single answer (A/B/C/D) |

### Domains (with blueprint weights)

| ID | Domain | Weight |
|----|--------|--------|
| d1 | Agentic Architecture & Orchestration | 27% |
| d2 | Tool Design & MCP Integration | 18% |
| d3 | Claude Code Configuration & Workflows | 20% |
| d4 | Prompt Engineering & Structured Output | 20% |
| d5 | Context Management & Reliability | 15% |

The mock exam composes its 60 questions using these weights.

## Commands

```
uv run -m tutor study              # adaptive flashcards
uv run -m tutor study --reset      # wipe history first, then study
uv run -m tutor exam               # timed 60-question mock
uv run -m tutor stats              # performance breakdown
uv run -m tutor refs               # show reference URLs
uv run -m tutor reset              # erase all study history (with confirmation)
```

### `study` — adaptive flashcards

A loop that picks a question, asks it, scores it, shows the explanation, and continues until the user types `Q`.

**Question selection** each turn:
1. With probability ~0.4, pull from the retry queue if anything is *eligible* (see cooldown below). Prefer eligible entries whose domain is currently weak.
2. Otherwise pull a fresh, unasked question from the bank, biased toward weak domains when possible.
3. Otherwise pull any unasked question.
4. Otherwise the bank is exhausted — print a "you've seen them all" message and exit cleanly.

**Choice shuffling.** Every time a question is served, randomly remap A/B/C/D so the answer letter isn't memorized. Update `correct_answer` to match.

**Scoring & retry queue.** Each answer prints a colored result panel (green correct / red incorrect with the right answer highlighted) plus the question's `explanation`, paragraph-broken so each option's analysis sits on its own line. Wrong answers go on a retry queue with a **5-question cooldown** — they cannot resurface as the next question or any of the next four. Repeat misses bump the cooldown again.

**Weak domains.** A domain is "weak" when accuracy in the most recent N answers (default 30) falls below threshold (default 0.5). Drives both seed and retry selection. Operates on **domains** (the five CCA-F domains), not on individual task statements.

**Header (always visible at the top of the screen):**
- A pass-probability gauge (logistic on per-domain accuracy, blended with a 0.5 prior until enough data accumulates)
- Overall correct/asked/percent
- Per-domain correct/asked, abbreviated (`Agentic`, `MCP/Tools`, `Claude Code`, `Prompting`, `Context`)

**After a missed question**, also print a "Further reading" panel listing only the references relevant to that question's domain (see References below).

### `exam` — timed mock

Mirrors the real exam: 60 questions sampled by domain weight, 120-minute countdown, no explanations until the end. `S` skips a question (returned to at the end), `Q` finishes early. A live timer ticks at the top of the terminal.

After completion: pass/fail screen, per-domain breakdown, and a list of every missed question with its correct answer and explanation. Misses get appended to the retry queue so they show up in subsequent `study` sessions.

**Critical:** exam answers must **not** flow into the per-domain accuracy stats that drive `study` selection or the pass-probability gauge. Conflating mock-exam pressure with study practice corrupts both signals. Track exam runs in a separate `exam_runs` log on the session.

### `stats`

Show:
- Overall correct/asked
- Per-domain table (asked, correct, rate)
- Per-task-statement table (informational only — does not drive selection)
- Retry queue size
- Recent study runs (last 5)
- Recent exam runs (last 5)

### `refs`

Print all configured reference URLs (the four Partner Network category folders and the local exam-guide PDF). The same per-domain-filtered subset surfaces under each missed `study` question.

### `reset` / `study --reset`

Confirms with `yes`/anything-else, then deletes `data/session.json`. `study --reset` does the wipe and then immediately enters the study loop.

## References

`config.yaml` carries reference links:

```yaml
references:
  exam_guide: "EXAM_GUIDE.pdf"   # bundled in the repo
  action: "https://drive.google.com/..."   # Partner Network: Claude Code
  api:    "https://drive.google.com/..."   # Partner Network: Claude API
  mcp:    "https://drive.google.com/..."   # Partner Network: MCP
  skills: "https://drive.google.com/..."   # Partner Network: Skills
```

The exam guide PDF is committed to the repo because it's publicly available on Anthropic's CCA page. The four Partner Network category folders are *not* redistributed — they're behind a shared Google Drive that only Partner Org members can open.

**Domain → reference mapping** (filters what shows after a missed `study` question):

| Domain | Categories shown (plus exam guide) |
|---|---|
| d1 Agentic Architecture & Orchestration | api, action |
| d2 Tool Design & MCP Integration | mcp, api |
| d3 Claude Code Configuration & Workflows | action, skills |
| d4 Prompt Engineering & Structured Output | api |
| d5 Context Management & Reliability | api, action |

Local file paths (like `EXAM_GUIDE.pdf`) get resolved to absolute paths on display so iTerm / similar terminals can ⌘-click them.

## Storage

Single JSON file at `data/session.json`, gitignored. Holds:

- `total_questions`, `total_correct`
- `domains` — `{ domain_name: { asked, correct } }` — **study answers only**, never exam
- `topics` — `{ task_statement: { asked, correct } }` — informational
- `history` — full per-answer log (study only)
- `asked_seed_ids` — list of question ids already served from the bank in `study`
- `retry_queue` — missed questions with `not_before` cooldown markers
- `session_runs` — per-`study`-session summaries
- `exam_runs` — per-`exam` summaries (score, total, elapsed, per-domain breakdown)
- `current_run_start` — timestamp marker for the in-progress study run

`Session.load` defaults missing keys; old saves shouldn't break.

## Tech & dependencies

- Python 3.14+
- [uv](https://docs.astral.sh/uv/) for env management
- `rich` (panels, tables, colors)
- `prompt_toolkit` (input)
- `pyyaml` (config)

That's it. No `openai`, `anthropic`, `chromadb`, or any network library.

## Non-goals (don't build these)

- **Difficulty levels** ("Beginner / Intermediate / Expert" tiers). The bank has no per-question difficulty signal; any level system would be theatre.
- **Multi-select questions.** The CCA-F is single-answer; don't plumb `multi_select` through the data model "just in case."
- **Case-study bundling.** The current bank has zero case-study questions. Don't reserve slots for them in exam composition.
- **Per-concept ("products") tracking.** The new bank doesn't carry consistent concept tags. Track at domain + task-statement level only.
- **LLM/RAG/ingestion.** No embedding, no vector store, no API calls. The whole point is offline.
- **Multi-cert support.** Single cert (CCA-F). Don't generalize prematurely.

## Lessons from the previous iteration (mistakes to avoid)

- **Weak-spot detection silently broken.** The prior code returned task-statement codes (`"1.1"`, `"1.2"`...) but compared them against full-name domains (`"Tool Design & MCP Integration"`). Match was always empty; weak-spot bias was a no-op. Operate on domains end-to-end.
- **Exam answers polluted study stats.** `cmd_exam` called the same `record()` as `cmd_study`, so mock-exam answers fed into the per-domain accuracy that drives study selection and the pass gauge. Keep them separate.
- **Dead `products` plumbing.** A "Weak Concepts" table that was always empty after the bank schema change. Don't ship features that depend on optional fields.
- **Retry queue had no cooldown.** Missing a question at total_questions=N could pull it again at N+1. Use `not_before = total_questions + 5` on insert; gate `get_retry_question` on eligibility.
- **Filename misspellings stick.** `claud_certified_architect_question_bank.json` lived for a couple commits. Pick names carefully on day one.
- **Header panels with `height=3`** clip awkwardly when the content wraps. Let panels size naturally.
- **`config.yaml` resolved relative to cwd.** Made the tool brittle to invocation directory. Resolve relative to the file or repo root instead.
